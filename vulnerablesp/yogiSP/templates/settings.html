Important: Signature Presence vs. Validation

wantMessagesSigned / wantAssertionsSigned only check that a <Signature> XML element is present. They do NOT verify the signature cryptographically. An attacker can modify the SAML response content and the existing (now-invalid) signature element will still satisfy these checks.

validMessage / validAssertion perform actual cryptographic validation of the signature. When enabled, any modification to the signed XML will be detected and rejected.

Real-world CVE-2017-11427 demo: Enable validMessage or validAssertion + CVE-2017-11427, then log in as booboo (group: administratorsnot). Intercept the SAML response and change administratorsnot → administrators<!---->not. XML C14N strips comments so the signature stays valid, but the vulnerable parser reads administrators → privilege escalation! Direct administrators fails because it doesn't match the original signed value.
