Vulnerability Configuration

Toggle attack surface simulations on/off. OOB requests (XXE/XSLT) are restricted to *.oastify.com only. XSLT also allows local file read.

XSW XML Signature Wrapping
When enabled, attackers can use XML Signature Wrapping (XSW1–XSW8) to bypass schema validation and signature verification, escalating privileges.
XXE XML External Entity
When enabled, the XML parser resolves external entities (DTD), allowing attackers to inject <!DOCTYPE> declarations that trigger out-of-band requests. Only *.oastify.com domains are reachable.
XSLT XSLT Injection
When enabled, XSLT stylesheets embedded in <ds:Transform> elements are processed during signature verification. Allows local file read (/etc/passwd etc.) and OOB requests to *.oastify.com.
More vulnerability modes will be added in future updates.
CVE Configuration

CVE-2017-11427 XML Comment Injection
When enabled, attackers can inject XML comments into signed attribute values to bypass signature validation and escalate privileges.
CVE-2022-41912 Multiple Assertion Bypass
When enabled, the SP accepts SAML Responses containing multiple Assertion elements. It validates the signature on the first (signed) Assertion but reads user data from the last (unsigned, attacker-injected) Assertion, allowing authentication bypass.
